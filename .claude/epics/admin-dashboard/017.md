# Task 001: Limpiar referencias al dashboard del sitio público

## Objetivo

Eliminar todos los botones, links y componentes relacionados con `/dashboard`, `/auth/login` y `/auth/register` del sitio público. El visitante de la landing no debe ver ningún CTA de "iniciar sesión" o "dashboard".

## Archivos a Modificar

### 1. `src/components/common/header.tsx`
- Eliminar import de `HeaderDashboardCTA` y `DashboardCTAButton`
- Eliminar `<HeaderDashboardCTA>` del header desktop
- Eliminar el botón de "Dashboard" / "Iniciar Sesión" del menú mobile
- El header solo debe tener: logo + navegación (casos de uso, FAQ, precios) + CTA de "Solicitar Demo" (que va al formulario de contacto de la landing)

### 2. `src/components/common/dashboard-cta-button.tsx`
- Marcar como deprecated o eliminar el archivo completo
- Si algún componente lo importa y no es el header, también hay que limpiarlo

### 3. `src/app/page.tsx`
- Verificar si hay algún import o uso de `DashboardCTAButton` o links a `/dashboard`
- Si existe el `FloatingDashboardCTA`, eliminarlo

### 4. `src/components/landing/hero-section.tsx`
- Verificar si el hero tiene botón de dashboard/login
- Si hay CTA secundario que va a `/dashboard`, cambiarlo a ancla al formulario de contacto

## Verificación

```bash
# Después de los cambios, ninguno de estos debe aparecer en archivos de src/
grep -r "dashboard-cta-button\|HeaderDashboardCTA\|HeroDashboardCTA\|FloatingDashboardCTA\|/auth/login\|/auth/register\|/dashboard" src/app/page.tsx src/components/common/ src/components/landing/
```
